Эссе на тему выбора Erlang/OTP в рамках курса «Функциональное программирование»

Я выбрал Erlang/OTP. Во-первых, он создан для параллельных задач: множество лёгких процессов и обмен сообщениями позволяют нагружать все ядра без общей памяти и гонок. Во-вторых, в языке заложена надёжность: есть механизмы перезапуска упавших процессов и работа «без простоя», включая горячую замену кода. В-третьих, OTP даёт готовые шаблоны для типовых компонентов и чёткую структуру проекта, что ускоряет разработку и упрощает поддержку. Эти свойства напрямую полезны для учебных работ по ФП и последующих дисциплин.

Проект для 4-й работы — простая модель нейросети. Каждый «нейрон» — отдельный процесс. «Синапсы» — сообщения между ними. Можно показать отказоустойчивость, отключая часть узлов и наблюдая, как сеть продолжает работу. Дополнительно — менять конфигурацию сети на лету и фиксировать поведение. Такая постановка естественно ложится на процессы и сообщения Erlang и демонстрирует сильные стороны платформы.

Для выполнения работ использую актуальный Erlang/OTP 28. Сборка — rebar3. Форматирование — erlfmt. Статический анализ — Dialyzer. Линтер — Elvis. Тесты — EUnit для модульных и Common Test для интеграционных. В CI — сборка, тесты, автоформатирование, линтер, статический анализ и запрет предупреждений. Репозиторий — чистая история коммитов и понятная структура каталогов.

Использованные и планируемые материалы:

* Fred Hébert — *Learn You Some Erlang for Great Good!*
* Joe Armstrong — *Programming Erlang: Software for a Concurrent World*
* Francesco Cesarini, Steve Vinoski — *Designing for Scalability with Erlang/OTP*
* Документация Erlang/OTP, EUnit, Common Test, rebar3, Dialyzer, Elvis, erlfmt
